1. Overview

This app drives the ProtoArc T1 Plus Bluetooth touchpad from user space (no DriverKit, no SIP changes). It matches the device with IOHIDManager (Vendor ID 1256 / 0x04E8, Product ID 28705 / 0x7021), reads raw HID digitizer reports, and posts mouse/scroll/keyboard events via CGEvent.

2. Architecture & data flow

Bluetooth HID  →  IOHIDManager (dedicated CFRunLoop thread: HIDRunLoop)
                      ↓ input report callback
                 TouchpadController.handleReport
                      ↓ ReportParser → TouchFrame
                 GestureEngine.process(frame:)
                      ↓
                 EventSynthesizer  →  CGEvent (cursor, clicks, scroll, shortcuts)

SwiftUI (main actor): MenuBarExtra, Settings window, TouchpadUIState for live view

3. How to build and run

Requirements

  • macOS with Xcode 26+ (project targets macOS 26 SDK).
  • Apple Silicon or Intel Mac; signing can be Sign to Run Locally / Automatic.

Build in Xcode

  1. Open ProtoArc T1 Plus.xcodeproj.
  2. Select scheme ProtoArc T1 Plus and destination My Mac.
  3. Press ⌘R (Run) or ⌘B (Build).

Build from Terminal

cd "/path/to/ProtoArc T1 Plus"
xcodebuild -project "ProtoArc T1 Plus.xcodeproj" \
  -scheme "ProtoArc T1 Plus" -configuration Debug build

Output app: ~/Library/Developer/Xcode/DerivedData/.../Build/Products/Debug/ProtoArc T1 Plus.app

First run checklist

  • Grant Input Monitoring and Accessibility in System Settings → Privacy & Security.
  • Activate a serial from ProtoArc T1 Plus/SerialNumbers.txt (bundled with the app).
  • Pair the touchpad over Bluetooth; press Start Driver in the menu bar or Settings.

4. Permissions & licensing

  • Input Monitoring — required to receive HID reports from the device.
  • Accessibility — required to synthesize pointer and keyboard events.
  • The app target is not sandboxed (needed for HID + event injection).
  • SerialManager loads allowed serials from SerialNumbers.txt: weekly (7 days after activation) and active (permanent).

5. Source files explained

Primary sources live in the synchronized group ProtoArc T1 Plus/ (Xcode target). Root-level duplicates like ContentView.swift at repo root are not the main app target unless explicitly added.

FileRole
ProtoArc_T1_PlusApp.swift@main SwiftUI app: creates TouchpadController, wires AppDelegate, defines MenuBarExtra, Settings window, and License window.
AppDelegate.swiftNSApplication delegate: sets activation policy to accessory (menu bar app), activates app on launch, runs AppStartup and unstickPointer. Terminating after last window closed is disabled so the driver keeps running.
MenuBarView.swiftMenu bar UI: start/stop, settings, license gate, quit. AppStartup syncs launch-at-login and optionally auto-starts the driver if licensed.
ContentView.swiftSettings window layout: sidebar (status, permissions, license, toggles) and live touch panel; forwards logging/layout changes to the controller for HID-thread caches.
LicenseView.swiftLicenseSection in settings and LicenseGateView for activation UI bound to SerialManager.
TouchpadController.swiftCentral coordinator: license check, permissions, IOHIDManager open/close on HIDRunLoop, device match/removal, report callback → ReportParser + GestureEngine; publishes UI state on main actor; report rate timer on main.
HIDRunLoop.swiftDedicated thread running a CFRunLoop so HID callbacks never block the main thread (reduces UI freezes).
ReportParser.swiftParses Report ID 2 (digitizer) and mouse report into TouchFrame / button bitfields; ReportLayout switches buffer layout (report ID stripped vs included).
GestureEngine.swiftState machine: pointer movement, taps, multi-finger scroll, pinch zoom shortcuts, three-finger swipes, physical button handling, double-tap-drag window (timing constants near top of file).
EventSynthesizer.swiftPosts CGEvent for move/drag, clicks, scroll wheel, and system gesture key chords; uses a lock because it may be called from the HID thread.
TouchModels.swiftDeviceIDs, TouchContact, TouchFrame, enums for tap/gesture actions, and TouchpadSettings (UserDefaults-backed toggles and sliders).
TouchpadUIState.swiftObservable UI-only state: throttled live finger display, raw report log when enabled; isolated from menu bar updates to limit SwiftUI churn.
Permissions.swiftChecks and prompts for Input Monitoring and Accessibility (AXIsProcessTrusted).
LaunchAtLogin.swiftRegisters the app as a login item via SMAppService when the user enables launch at login.
SerialManager.swiftLoads SerialNumbers.txt, validates weekly vs active serials, stores activation in UserDefaults.
SerialNumbers.txtCatalog of allowed serials ([weekly] / [active] sections). Edit and rebuild to change keys.
TouchpadManager.swiftHistorical / reference comments or helpers (not the primary HID path); see file header in project.

DriverKit files at repo root (ProtoArcTouchpadDriver.*, ExtensionManager.swift) are reference only and are not linked into the user-space app target.
